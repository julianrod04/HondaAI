## Environment Setup

**Primary venv:** `./venv` (Python 3.12)

```bash
# Activate
source venv/bin/activate          # Linux/Mac
venv\Scripts\activate.bat         # Windows

# Install dependencies
pip install -r requirements.txt
```

**Key packages:** carla==0.9.16, stable_baselines3==2.7.1, gymnasium==1.2.3,
torch==2.10.0, pygame==2.6.1, opencv-python, tensorboard, pandas, numpy

> **RTX 5080 (Blackwell) note:** Requires PyTorch 2.7+ with CUDA 12.8.
> Current torch 2.10.0 doesn't support it — use `--cpu` flag for training.

## Running Things

### Start CARLA server
```bash
cd CARLA-sim/CARLA_0.9.16
./CarlaUE4.sh                                          # Linux
# CarlaUE4.exe                                         # Windows
```
Or use the GUI launcher: `python3 launcher.py`

### Train RL agent
```bash
source venv/bin/activate
cd CARLA-sim/CustomPython
python scripts/train.py --num-envs 4 --timesteps 1000000 --light
python scripts/train.py --cpu          # CPU-only (RTX 5080 workaround)
```

### Launch multiple CARLA servers (parallel training)
```bash
python scripts/launch_servers.py --num-instances 4
python scripts/launch_servers.py --stop
```

### Manual driving
```bash
python CARLA-sim/CustomPython/kw_sandbox/manual_drive.py
```

### Monitor training
```bash
tensorboard --logdir CARLA-sim/CustomPython/tb_logs
```

## GUI Launcher (`launcher.py`)

Named **HondaAI-CARLA-Launcher**. Built with Tkinter (pure stdlib, no PyInstaller).
Config stored as `.hondaai_launcher_config.json` next to the launcher.

Run: `python3 launcher.py` (Linux) or double-click `launcher.pyw` (Windows).

### Tabs

1. **Setup** — Configure CARLA root + venv path with live validation; auto-detect CARLA; create venv. Shown first when config is invalid.
2. **CARLA Server** — Start/stop CARLA, process table
3. **Virtual Env** — Create/delete venv, pip management
4. **Scripts** — Browse and launch Python scripts
5. **Terminal** — Embedded shell with history and color output

### CARLA Server tab details
- Unified process table showing ALL launcher-managed processes (CARLA + scripts + shell commands)
  - Columns: Type, Name, PID, Status, Started, Command
  - Also shows system-level CARLA processes not launched by the GUI (via `ps`/`tasklist`)
  - Auto-refreshes every 3 seconds
  - Kill Selected, Kill All Scripts, Remove Exited buttons

### Tab 2 — Virtual Env

- Create / delete venv at any path
- Shows Python version and active status
- Activation command displayed with copy-to-clipboard button
- Requirements file dropdown (pre-populated with known candidates under CARLA root)
- Browse for any `requirements.txt`
- Install requirements, pip freeze, pip list
- Install / uninstall individual packages by name

### Tab 3 — Scripts

- Default source: `CARLA_x.x.x/PythonAPI/examples/` (all `.py` files loaded automatically)
- Add individual script files via file picker
- Add entire directories of scripts (all `.py` files scanned)
- Custom scripts and directories persisted in `launcher_config.json`
- Treeview with Source / Name / Path columns; double-click to launch
- Args field for passing CLI arguments to any script
- Remove Selected removes custom entries (cannot remove built-in examples dir entries)
- All scripts run using the selected venv's Python
- All launched scripts appear in the unified process table

### Tab 4 — Terminal

- Scrollable output area with color-coded sources:
  - Cyan = CARLA output
  - Green = script output
  - Yellow = shell commands
  - Red = errors
  - Blue/bold = typed commands
- Shell input bar at the bottom with `(venv-name) $` prompt
- Commands run with venv activated (venv `bin/` prepended to PATH)
- Up/Down arrow key command history
- Tab completion for file paths
- Save Log to file button
- Clear button
- All process output (CARLA, scripts, pip, shell) streams here in real time

## RL Configuration (`rl/config.py`)

All hyperparameters are dataclasses — edit there, not scattered in scripts.

- **Map:** Town06, 200m straight highway segment
- **Spawn:** (21, 244.49) → Goal: (221, 244.49)
- **Vehicle:** Dodge Charger
- **Observation:** 12-dim state vector + optional 84×84 RGB camera
- **Action space:** continuous [throttle/brake, steering] in [-1, 1]
- **CARLA ports:** base 2000, +2 per instance (2000, 2002, 2004, 2006)

## Alert System (`kw_sandbox/alerts.py`)

- `DrivingMonitor` — detects lane drift, speeding, proximity violations
- `AlertType` enum — DRIFTING_LEFT/RIGHT, SPEED_UP/DOWN, TURN_LEFT/RIGHT, STOP, EMERGENCY_STOP, SPEEDING, PROXIMITY_WARNINGS
- Two overlay panels: FOV (450×80px center) + Dashboard (bottom-right banner)
